## 难度校准结果

### 第 1 关

- 目标定位：教学型，先保证玩家能理解“看红眼 -> 拖镜头 -> 屏息 -> 开枪”
- 当前配置：
  - 目标 `3`
  - 市民 `2`
  - 时限 `90s`
  - 奖励 `80`
  - 道具：`scan = 1`，`time_extend = 0`
- 当前落地点：
  - 三个目标全部为 `static`
  - 三个目标识别强度由易到难递增
  - 市民伪装强度略高于教学主目标，避免一上来误导过猛

判断：
- 这一关的重点不是压时间，而是降低第一次理解门槛。
- 所以把 `time_extend` 先关掉，只保留 `scan`，避免玩家在第 1 关把学习重点放到“补时间”而不是“识别目标”上。

### 第 2 关

- 目标定位：让玩家明显感受到“目标不再站着等你”
- 当前配置：
  - 目标 `4`
  - 市民 `3`
  - 时限 `100s`
  - 奖励 `110`
  - 道具：`scan = 1`，`time_extend = 0`
- 当前落地点：
  - `2` 个静止目标
  - `2` 个移动目标
  - 第一个移动目标位移更短、速度更低
  - 第二个移动目标位移更长、速度更高，且放在更容易和干扰项混淆的位置

判断：
- 第 2 关的压力主要来自“观察后还要跟枪”，不是来自时限爆炸。
- 所以这里仍然不开放 `time_extend`，让失败原因更聚焦在“观察/跟枪”本身，而不是靠补时间兜过去。

### 第 3 关

- 目标定位：第一次逼出“先打谁、何时用道具”的决策
- 当前配置：
  - 目标 `5`
  - 市民 `4`
  - 时限 `110s`
  - 奖励 `150`
  - 道具：`scan = 1`，`time_extend = 1`
- 当前落地点：
  - `2` 个静止目标
  - `1` 个移动目标
  - `2` 个弱点显露目标
  - 两个弱点目标的显露周期不同，其中一个窗口更短

判断：
- 第 3 关不再继续堆移动目标数量，而是把压力转向“优先级 + 窗口判断”。
- 这样更符合文档里“玩家开始愿意主动使用道具”的预期，不会变成单纯的人海战术。

## 贴近业务的测试建议

这轮建议直接按下面顺序验，不要只看“功能有没有按钮”。

1. 第 1 关新手验证
- 找 1 名第一次接触项目的人试玩
- 看他是否能在 `30 秒` 内完成第一枪
- 看他是否会误把普通市民当目标乱打

2. 第 2 关节奏验证
- 连续从第 1 关打到第 2 关
- 记录第一次失败原因是：
  - 找不到目标
  - 看得到但打不中
  - 时间不够
  - 误伤过多

3. 第 3 关道具验证
- 重点记录：
  - 是否主动按过 `scan`
  - 是否在剩余时间紧张时按过 `time_extend`
  - 是否卡在弱点窗口判断上

4. 结算驱动验证
- 通关 2 到 3 局
- 看玩家是否会：
  - 点击升级
  - 主动进下一关
  - 觉得奖励差异“看得出来”
