# Step 7 教程落地说明

## 本次落地范围

本次把 `Step 7 教程` 直接接入到现有 `PVE` 主流程里，目标是让玩家第一次进入第 1 关时，不需要额外口头说明，也能按步骤完成第一枪。

当前教程只在 `第 1 关首次进入` 时触发，后续同一轮进度内不会重复强制教学。

## 已实现内容

1. 新增教程状态管理
2. 新增教程引导场景
3. 战斗场景接入教程锁步
4. 菜单页增加教程状态提示
5. 教程开始/完成写入埋点与存档 payload

## 当前教程步骤

1. 移动镜头：`WASD` 或方向键
2. 缩放观察：`Q / E` 或鼠标滚轮
3. 屏息稳定：`Shift`
4. 使用扫描：`1`
5. 开出第一枪：`Space / Enter`

规则：

- 当前步骤未完成前，后续关键操作不会提前放开
- 玩家如果提前按了后续功能，界面会直接提示“先完成当前教程步骤”
- 教程完成后，道具和战斗操作全部恢复为自由状态

## 贴近业务的验证方式

### 场景 1：首次进入第 1 关

1. 从主菜单点击 `开始第 1 关`
2. 确认左上角出现教程引导面板
3. 先按 `Space` 测试
4. 预期：不会提前放枪，并提示先完成当前步骤

### 场景 2：按顺序完成教学

1. 按 `WASD` 移动镜头
2. 按 `Q / E` 缩放
3. 按住 `Shift`
4. 按 `1` 使用扫描
5. 按 `Space`

预期：

- 每完成一步，教程文案推进到下一步
- 最后一步完成后显示“教程完成”
- 此后 `2` 时间延长和其他战斗操作恢复可用

### 场景 3：完成后重开同关

1. 完成教程
2. 正常结束本局或返回主页后再次进入

预期：

- 菜单页显示 `教程状态：已完成`
- 当前会话内再次进入第 1 关时不再强制逐步教学

## 当前已知边界

- 目前“教程完成态”已写入存档 payload，但是否跨重启恢复，取决于后续是否把 mock 平台存档真正接回读取链路
- 现在教程挂在 `PVE` 主场景里，后续如果要扩展成独立教程关，可以把同一套步骤配置迁移到 `configs/tutorial/`
