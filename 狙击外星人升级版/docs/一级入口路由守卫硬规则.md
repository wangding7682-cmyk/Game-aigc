# 一级入口路由守卫硬规则

## 目的

解决“按钮点击后看起来没反应”的共性问题。

过去多次问题的根因都不是按钮本身失效，而是：

- 按钮点击后直接 `emit`
- 目标页在 `instantiate()`、`_ready()`、资源加载或存档恢复阶段报错
- 页面上没有任何失败反馈
- 用户最终只能感知为“点了没反应”

为避免同类问题重复出现，项目内所有一级入口统一执行本规则。

## 适用范围

以下入口全部属于一级入口：

- 主菜单进入其他页面
- 结算页进入升级页、下一关、重开、返回主页
- 升级页进入下一关、返回主页
- 设置页、调参页、武器库、商店、测试中心的返回主页
- 战斗页 / PVP 页的返回主页
- 联机大厅进入联机房间

## 硬性规则

### 规则 1：禁止盲跳转

所有一级入口按钮，禁止直接写：

```gdscript
CoreEventBus.xxx_requested.emit()
```

必须统一走：

```gdscript
RouteGuard.request_route(route_id, origin, payload)
```

### 规则 2：点击后必须二选一

一级入口点击后，`500ms` 内必须满足以下二选一结果之一：

1. 目标页成功打开
2. 页面出现明确错误提示

禁止出现“无反馈、无跳转、无错误提示”的静默失败。

### 规则 3：目标页显示成功必须回执

所有通过 `core_game_root.gd` 切换的页面，在 `_show_scene()` 完成后，必须调用：

```gdscript
RouteGuard.confirm_scene(current_screen.scene_file_path)
```

用于给路由守卫返回“页面已成功打开”的确认。

### 规则 4：失败必须对用户可见

如果目标页未在 `500ms` 内成功打开，必须显示明确提示，例如：

- `打开商店失败：目标页未在 500ms 内就绪，请查看日志。`
- `跳转失败：未定义入口 test_xxx。`

禁止只在控制台打印日志，而界面无任何提示。

### 规则 5：一级入口必须有 smoke test

所有一级入口相关跳转，必须至少被以下之一覆盖：

- 主流程烟雾测试
- 下一关烟雾测试
- 完整集成烟雾测试
- 路由守卫烟雾测试

新增一级入口时，必须同步补充测试覆盖。

## 验收标准

任一一级入口完成后，必须同时满足：

- 点击后不会出现静默无响应
- 成功时能进入正确页面
- 失败时能显示明确提示
- `RouteGuard` 有对应路由定义
- 测试中心可复测，或已有 smoke test 覆盖

## 开发约束

新增一级入口时，开发顺序固定为：

1. 在 `core_route_guard.gd` 中注册路由
2. 在入口按钮处接入 `RouteGuard.request_route(...)`
3. 确认 `core_game_root.gd` 能回执成功页面
4. 补充 smoke test
5. 在测试中心中可被复测

如果以上任一项缺失，则视为未完成。
