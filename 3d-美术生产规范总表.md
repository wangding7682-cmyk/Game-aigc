# 《3D 美术生产规范总表》

> 版本：2026-06-30 校准版  
> 文档定位：当前项目 3D 生产基线，服务 `Godot 3D MVP` 主链路。它不负责商业化页面和扩张型系统的视觉细则，重点约束角色、武器、场景、特效、贴花、碰撞和挂点如何做成“能接、能读、能验证”的资产。  

---

## 文档范围

这份文档现在只管 8 类 3D 生产问题：

1. 模型
2. 材质与贴图
3. 骨骼与动作
4. 世界空间特效
5. 场景模块
6. 贴花与命中痕迹
7. 碰撞
8. 挂点

它不直接负责：

- HUD 组件视觉规范
- 商店、武器库等页面级组件规范
- Battle Pass、赛季、段位等扩张系统的专门生产规则

这些内容如果后续重新进入主范围，应单开扩展文档，不再混在当前 3D 生产基线里。

---

## 当前生产目标

当前项目的 3D 生产，不是为了先把资产做得最华丽，而是为了稳定支撑这条业务链路：

`搜索识别 -> 放大瞄准 -> 开火判断 -> 命中 / 误判 / 挡弹反馈 -> 复盘与继续搜索`

所以 3D 生产的总目标必须同时满足：

1. 身份可读
2. 状态可读
3. 遮挡可读
4. 命中可读
5. 性能可控

当真实感和玩法可读性冲突时，统一优先级如下：

1. 玩法可读性
2. 身份差异
3. 状态差异
4. 命中与反馈清晰度
5. 画面真实感
6. 装饰性细节

## 资产分类

| 大类 | 当前作用 |
|---|---|
| 角色资产 | 外星人与平民的识别、误判与弱点窗口 |
| 武器资产 | 第一人称阅读、开火语义与挂点承载 |
| 场景资产 | 搜索空间、遮挡关系与挡弹语义 |
| 特效资产 | 扫描、命中、误伤、挡弹、慢动作结果表达 |
| 贴花资产 | 子弹痕迹与表面命中反馈 |
| 绑定资产 | 碰撞、挂点、焦点、回放定位 |

## 模型规范

### 通用原则

| 项目 | 规范 |
|---|---|
| 模型优先级 | 先保证轮廓清晰，再补细节 |
| 体块表现 | 软 3D 卡通化体块为主，不走重写实路线 |
| 轮廓要求 | 中远景也能从轮廓识别对象类别 |
| 结构要求 | 结构服务玩法，而不是只追求复杂 |
| 层级要求 | 主结构、次结构、细节结构分明 |

### 通用禁止项

- 不用高频细节代替大轮廓差异
- 不允许近景好看、远景不可读
- 不允许为了“酷”牺牲身份识别
- 不允许模型造型过度写实，导致与当前镜头和 HUD 风格脱节

### 业务化验证

模型首次导入后至少回答：

1. 中远景能不能一眼看出它是什么类别
2. 掩体边缘露出时，仍能不能读出角色是否可疑
3. 武器放到第一人称视角时，会不会挡住准星与关键视区

## 角色规范

### 外星人

外星人必须在伪装前提下保留可疑特征，但不靠复杂表演暴露身份。

当前优先保留的稳定识别特征：

1. 头部外壳过紧
2. 肩线异常或肩袖连接偏硬
3. 体态略僵硬
4. 弱点可开启
5. 扫描时异常特征增强

### 平民

平民的目标不是“毫无信息”，而是“稳定安全”。

| 项目 | 规范 |
|---|---|
| 姿态 | 自然、平衡、呼吸稳定 |
| 肩线 | 平稳，不能持续异常 |
| 服装 | 蓝灰或中性色优先 |
| 假线索 | 可局部短时误导，但不能做成第二种外星人 |

### 角色生产必备项

每个角色资产至少应具备：

1. 显示模型
2. 主命中区
3. 头部或上半身识别重点
4. 命中焦点
5. 状态挂点

### 业务化验证

建议至少做 3 个场景验证：

1. 新玩家首次进入首关，能否逐步学会区分真线索和假线索
2. 误判回放里，玩家能否看懂为什么这是平民
3. 同一角色在远、中、近景下的判断是否一致

## 武器规范

### 生产原则

| 原则 | 说明 |
|---|---|
| 剪影优先 | 不同枪型先靠轮廓区分 |
| 视角优先 | 第一人称展示角度必须稳定 |
| 主题强化 | 颜色和发光结构强化气质，不替代剪影 |
| 皮肤从属 | 外观变化不能破坏本体识别 |

### 当前四类武器关键词

| 武器 | 关键词 |
|---|---|
| 标准狙击枪 | 基础、军用、稳重 |
| 精准狙击枪 | 精密、战术、蓝色强调 |
| 连发狙击枪 | 紧凑、进攻、烈焰红 |
| 电磁狙击枪 | 未来、能量、等离子绿 |

### 武器输出层级

当前最少准备：

1. 第一人称战斗版
2. 简化远景版或替代方案

“展示版”可以保留为后续扩展需求，但不作为当前主生产验收的强依赖。

### 禁止项

- 不能只靠贴图区分武器
- 不能让瞄准镜与枪身结构遮挡核心视野
- 不能让高科技发光结构压过准星和命中反馈

### 业务化验证

至少验证：

1. 搜索态切换到瞄准态时，武器显示是否稳定
2. 开火瞬间，枪口反馈是否清楚但不遮挡结果阅读
3. 第一次接触项目的人能否一眼看出四类枪型差异

## 材质与贴图规范

### 材质基调

整体方向：

- 轻科幻
- 非写实
- 柔和高光
- 中低镜面
- 软 3D 卡通表达

### 材质分类

| 材质类型 | 规范 |
|---|---|
| 角色皮肤 / 服装 | 中低反射，优先可读性 |
| 武器主体 | 允许适度金属感，但保持风格统一 |
| 弱点 / 能量结构 | 可用 `emission`，但必须节制 |
| 城市场景件 | 服务遮挡与识别，不做夸张镜面 |
| 掩体 | 可更厚重粗糙，强化挡弹语义 |

### 材质禁止项

- 不允许高镜面泛白导致信息丢失
- 不允许平民材质比外星人更抢眼
- 不允许所有材质都发光
- 不允许场景材质过亮导致角色淹没

### 贴图原则

| 项目 | 规范 |
|---|---|
| Base Color | 负责主色区分，不能承担全部细节表达 |
| Normal | 只强化结构，不制造虚假轮廓 |
| Roughness | 控制材质气质，避免全都一样 |
| Emission | 只用于弱点、扫描和高科技结构 |
| 贴图密度 | 优先中远景可读，不追求极端细节 |

### 建议贴图尺寸

| 资产类别 | 建议级别 |
|---|---|
| 第一人称武器 | 2K 起步 |
| 角色主体 | 1K~2K |
| 小型场景件 | 512~1K |
| 中型掩体 / 货车 / 广告牌 | 1K~2K |
| 远景模块 | 可更低，优先性能 |

### 业务化验证

至少要确认：

1. 中景下角色不会被材质高光冲掉
2. 平民与外星人不会因为材质风格太近而混成一类
3. 弱点或能量结构不会在搜索态下过早暴露

## 骨骼与动作规范

### 骨骼目标

骨骼不是为了堆动作量，而是为了：

1. 支持状态切换
2. 支持识别差异
3. 支持命中反馈
4. 支持挂点逻辑

### 角色骨骼最低要求

每个角色至少支持：

1. 头部
2. 躯干
3. 双肩
4. 双臂
5. 髋部
6. 双腿

### 必须预留的重点

| 部位 | 原因 |
|---|---|
| 头部 | 头部朝向与识别重点 |
| 肩部 | 异常肩线核心 |
| 胸部 / 上躯干 | 弱点与扫描重点 |
| 髋部与腿部 | 移动态和重心变化 |

### 动作原则

动作要服务识别，而不是只服务表演。

### 当前动作基线

外星人建议优先支持：

1. 待机
2. 轻微异常待机
3. 横向移动
4. 扫描反应
5. 弱点开启
6. 被命中
7. 击杀收束

平民建议优先支持：

1. 普通待机
2. 普通行走
3. 受惊
4. 假线索瞬时反应

### 业务化验证

至少要确认：

1. 角色动起来后，身份差异不会完全消失
2. 弱点开启看起来像“时机窗口”，不是随机闪一下
3. 平民的动作不会越界成外星人第二形态

## 特效规范

### 特效分类

| 类型 | 用途 |
|---|---|
| 世界空间扫描 | 强化搜索逻辑 |
| 枪口火焰 | 强化开火反馈 |
| 命中火花 | 强化命中点反馈 |
| 掩体挡弹特效 | 强化 `blocked` 结果 |
| 弱点能量特效 | 强化时机窗口 |
| 慢动作 / 回放辅助特效 | 强化关键结果阅读 |

### 特效原则

1. 先讲结果语义，再讲风格
2. 世界特效不能和 HUD 特效抢中心视区
3. 特效不能影响角色身份读取
4. 误伤特效必须比普通 miss 更明确

### 颜色语义

| 类型 | 颜色 |
|---|---|
| 扫描 | 青蓝 |
| 命中 | 绿色 |
| 误伤 | 红色 |
| 掩体挡弹 | 冷蓝或中性亮色 |
| 慢动作 | 白蓝 |

### 业务化验证

至少要能回答：

1. 玩家开枪后能否立刻区分 `hit / wrong_hit / blocked / miss`
2. 命中与误伤反馈是否会遮住目标本身
3. 扫描反馈是否真的帮助搜索，而不是让画面更乱

## 场景模块规范

### 场景目标

场景模块既要能组成关卡，也要服务玩法。

### 当前模块分类

| 类型 | 说明 |
|---|---|
| 地面模块 | 街道、地砖、平台 |
| 掩体模块 | 墙角、箱体、货车、障碍板 |
| 街景模块 | 路灯、广告牌、围栏、城市家具 |
| 远景模块 | 天际线、建筑群、天空盒 |

### 场景要求

| 项目 | 规范 |
|---|---|
| 尺寸统一 | 模块尺度可拼装 |
| 语义明确 | 一眼看出是否挡弹、可穿行、可观察 |
| 性能优先 | 不做无意义高模堆叠 |
| 与角色分离 | 明暗和色相要让角色跳出来 |

### 掩体模块特殊要求

1. 能读出厚度
2. 射线逻辑明确
3. 遮挡比例合理
4. 命中后有清晰反馈

### 业务化验证

至少要确认：

1. 目标站在掩体边缘时，仍有合理识别机会
2. 被挡弹时，玩家能看懂不是打空
3. 新场景模块接入后，角色不会被背景吃掉

## LOD、碰撞与挂点

### LOD 目标

LOD 的目的不是只保性能，还要保不同距离下的身份识别。

### LOD 基本要求

| 资产类别 | 要求 |
|---|---|
| 目标角色 | 至少 2~3 级 |
| 第一人称武器 | 可不做常规远景 LOD，但要有战斗版与简化版区分 |
| 关键掩体 | 至少 2 级 |
| 远景建筑 | 必须简化 |

### LOD 规则

- 降级后不得丢掉关键识别轮廓
- 中景下外星人关键差异仍需可读
- 远景下平民与外星人不能变成同一种 silhouette

### 碰撞层

| 层号 | 层名 | 说明 |
|---|---|---|
| 1 | `world` | 地面 / 静态墙体 / 不可穿透建筑 |
| 2 | `target` | 外星人目标 |
| 3 | `civilian` | 普通市民 |
| 4 | `obstacle` | 可遮挡掩体 / 障碍物 |
| 5 | `interactive` | 可交互对象 |
| 6 | `bullet_decal` | 弹孔贴花，仅渲染不碰撞 |
| 7 | `pvp_actor` | 预留 PVP 单位层 |
| 8 | `scan_only` | 扫描检测专用层 |

### 碰撞要求

1. 目标和误伤对象必须分层
2. 掩体必须能正确挡射线
3. 贴花不参与碰撞
4. 弱点应有单独碰撞区

### 挂点目标

挂点用于连接：

- 命中反馈
- 特效播放
- 扫描高亮
- 弱点逻辑
- 回放镜头

### 当前关键挂点

角色必须预留：

- `HitFocus`
- `WeakPoint`
- `ScanHighlight`
- `HeadLook`
- `ReviewFocus`

武器建议预留：

- `Muzzle`
- `ShellEject`
- `Grip`
- `ScopeFocus`

场景件建议预留：

- `DecalSurface`
- `ImpactFx`
- `ScanFadeAnchor`

### 业务化验证

至少要确认：

1. 命中特效定位是否稳定
2. 弱点逻辑是否可单独控制
3. 回放镜头焦点是否稳定
4. 扫描层是否不干扰普通射击层

## 节点结构与命名

### 战斗场景固定结构

```text
BattleRoot3D
├─ LevelRoot
├─ ActorRoot
├─ DecalRoot
├─ FxRoot
├─ AudioRoot
└─ UiRoot
   ├─ HudLayer
   ├─ PopupLayer
   ├─ GuideLayer
   ├─ LoadingLayer
   └─ DebugLayer
```

### 节点命名

- 统一使用 `PascalCase`

常用示例：

- `AimCamera3D`
- `TargetSpawnPoint`
- `CivilianSpawnPoint`
- `WeakPointCollider`
- `DecalBulletHole`
- `PathPatrol`
- `FloatingText`

### 资源命名

| 类型 | 格式 |
|---|---|
| 模型 | `mdl_类别_名称_状态` |
| 材质 | `mat_类别_名称` |
| 贴图 | `tex_类别_名称_通道` |
| 动作 | `anim_角色_动作` |
| 特效 | `fx_语义_对象` |

## 一句话结论

当前 3D 美术生产层最重要的目标，不是先把所有资产做成最终形态，而是：`围绕 Godot 3D MVP 主战斗链路，把角色、武器、场景、特效、贴花、碰撞和挂点做成能接、能读、能验证、能逐步替换的稳定基线。`
