# 狙击外星人升级版

一句话卖点：在 3D 城市场景里“找不同”，用狙击镜把伪装目标从人群里揪出来。

这是一个以“识别伪装目标”为核心乐趣的 2.5D 识别射击项目。玩家会在 3D 城市场景里，从普通行人中找出伪装外星人，完成观察、放大、屏息、射击、复盘这一条完整闭环。

它不是一个只放程序代码的仓库，而是把玩法验证、Godot 主工程、AIGC 资产探索、UI 方案、工程化规范和回归资料都放在同一个地方，方便产品、策划、美术、程序一起对齐。

## 项目现在做到哪里

当前版本已经进入“可验证核心体验”的阶段，重点不是把所有商业化内容一次铺满，而是先把识别、瞄准、误判惩罚、反馈表现和关卡节奏做扎实。

目前仓库里能看到的重点包括：

- 以 `狙击外星人升级版` 为主的 Godot 4.x 可运行工程
- `3` 关 PVE 验证内容、教程接入、基础成长与结算方向
- PVP 大厅、房间、网络控制器骨架，暂时不是主验收重点
- 一整套 3D 占位资产、UI 占位稿、AIGC 美术试制与接入规范
- 围绕“设计是否真的落到工程里”的文档校准和回归资料

## 核心体验

这个项目想打磨的，不是传统“看见就开枪”的爽点，而是“先观察，再判断，再确认”的识别快感。

一局里的核心节奏是：

`扫视场景 -> 锁定可疑角色 -> 放大瞄准 -> 屏息稳定 -> 射击判定 -> 命中 / 误判 / 挡弹反馈 -> 继续搜索或进入结算`

在体验上，当前版本重点验证 5 件事：

- 玩家能不能快速读懂“谁可疑、为什么可疑”
- 放大与屏息之后，操作是否稳定、顺手
- 误伤平民时，反馈和复盘能不能让玩家马上明白错在哪里
- 扫描、加时、挡弹、命中这些反馈是否够清楚
- 设计文档、资源规范、工程实现能不能彼此对得上

## 快速开始

如果你只是想最快看到主工程，请从这里开始（对外开箱即跑版）：

1. 安装 `Godot 4.x`
2. 安装 `Git LFS`（用于拉取大体积的 3D 模型与音频资源）
3. 克隆仓库后执行一次 LFS 拉取：
   - `git lfs install`
   - `git lfs pull`
2. 用 Godot 打开 `狙击外星人升级版`
3. 载入后等待资源导入完成
4. 直接运行主工程，进入当前可验证版本

如果你克隆后发现模型/音效缺失，90% 的原因是没有安装或没有拉取 Git LFS（这时仓库里会是“指针文件”，Godot 无法当作真实资源加载）。

如果你更关心项目全貌，建议先看：

1. `GDD_狙击大师_完整设计方案.md`
2. `3d_asset_engineering/`
3. `sniper-game-asset-board/` 与 `sniper-master-mobile-ui/`
4. `project-execution-retro/`

## 协作建议

这个仓库既有主工程，也有文档、资产和验证资料，所以协作时最重要的是先对齐“这次改的是哪一层”。

- 改玩法或系统：优先进入 `狙击外星人升级版`
- 改资产接入规则：优先看 `3d_asset_engineering`
- 改展示图、预览页、批次资产说明：看对应的 `sniper-*` 目录
- 做版本复盘或交接：看 `project-execution-retro`

提交前建议至少做一轮贴近业务的简单确认：

1. 主工程能正常打开并进入场景
2. 一轮核心流程能跑通：观察、放大、射击、反馈、结算
3. 本次改动对应的文档或素材说明没有和工程口径脱节

## 路线图

这个项目的路线图按“先把核心体验做实，再扩展内容与平台”来排：

- 近期：继续打磨识别玩法的可读性与反馈语义，让玩家更清楚“为什么对/为什么错”；把 3 关 PVE 的节奏、难度与复盘做稳定
- 中期：把占位资产逐步替换为可量产的资产管线；完善教程、成长与装备系统，让“局内体验”和“局外动力”闭合
- 远期：按需要再把 PVP 骨架从“可用”推进到“可验收”，并评估多平台适配（移动端/小程序/短视频平台）

## Git 说明

仓库根目录已经补了适合上传 GitHub 的基础规则：

- 忽略 Godot 导入缓存和本地临时文件
- 忽略 `.env` 等本地环境信息
- 忽略 Python、Node、编辑器产生的缓存
- 为常见二进制资源补了基础属性，减少不必要的 diff 干扰
- `*.glb/*.wav` 等大资源使用 Git LFS 管理，保证“对外但开箱即跑”同时不让仓库膨胀失控

如果你准备第一次推到 GitHub，通常只需要在本地继续执行：

```bash
git lfs install
git add .
git commit -m "chore: initialize repository"
git remote add origin <你的仓库地址>
git push -u origin main
```
