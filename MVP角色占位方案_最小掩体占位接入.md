## 3. MVP 角色占位方案

## 3.1 外星人占位

### 目标
外星人必须看起来像“能混进城市人群里的可疑个体”，而不是纯怪物或纯人类。

### 当前实现特征

1. **服装化伪装层**
   - `CostumeMesh`
   - 用于表达都市潜伏感
   - 对应之前吸收的“服装化外星人”方向

2. **异常肩线**
   - 左右肩宽不完全对称
   - 在远景下也能形成隐约不自然的 silhouette

3. **红眼识别**
   - 目标角色显示眼部点
   - 通过红色脉冲增强“可疑性”

4. **弱点开启**
   - `WeakpointMesh`
   - 只在目标且弱点窗口出现时显示

5. **扫描高亮**
   - `HaloMesh`
   - 扫描时形成青蓝色外轮廓感

### 当前适合的外星人状态

1. 伪装待机
2. 服装化伪装待机
3. 移动态
4. 扫描高亮态
5. 弱点开启态

---

## 3.2 平民占位

### 目标
平民必须形成“安全对照样本”，同时允许存在轻度误导。

### 当前实现特征

1. 更稳定的肩线
2. 蓝灰 / 冷色服装基调
3. 默认无红眼
4. `AccentMesh` 可做假线索闪点
5. 扫描时允许有弱高亮，但不出现真正弱点结构

### 当前适合的平民状态

1. 普通待机
2. 普通移动
3. 假线索闪点态

---

## 3.3 当前角色占位的好处

比之前的胶囊体提升在于：

1. 玩家能开始读“头 / 肩 / 服装 / 胸口”
2. 外星人与平民开始有结构级差异，而不是只有颜色差异
3. 弱点和扫描高亮有单独结构
4. 更适合后续替换成正式模型，不会推翻当前逻辑

## 4. 最小掩体占位接入

## 4.1 掩体目标
这一步掩体不是为了做正式城市美术，而是为了验证：

1. 射线遮挡
2. 目标刷点与遮挡关系
3. 玩家能否看懂哪里可挡
4. 不同遮挡物是否形成不同干扰

## 4.2 当前接入方式

掩体由：

- `狙击外星人升级版/scripts/pve/pve_cover_obstacle_3d.gd`

负责生成；

并由：

- `狙击外星人升级版/scripts/pve/pve_battle_controller_3d.gd`

挂进：

- `LevelRoot`

### 当前优点

1. 掩体不再和角色混挂
2. 后续关卡占位和正式场景替换更容易
3. `FxRoot` / `DecalRoot` 可以单独承接命中特效与弹孔

---

## 4.3 四类最小掩体语义

### `wall_corner`
适合验证：

1. 固定遮挡
2. 边缘露头识别
3. 角落刷点

### `street_lamp`
适合验证：

1. 细长遮挡
2. 非完整遮挡
3. 中远景视觉干扰

### `parked_van`
适合验证：

1. 大型遮挡
2. 城市街区阻断
3. 路侧角色刷点

### `billboard`
适合验证：

1. 高位遮挡
2. 画面切割
3. 中景读取压力

## 7. 推荐测试顺序

### 先测角色

1. 外星人伪装态是否能看出“不是普通人”
2. 平民是否明显更安全
3. 弱点开启是否足够像窗口
4. 扫描高亮是否明显提高判断速度

### 再测掩体

1. 墙角是否最稳定挡弹
2. 路灯是否形成轻度误导
3. 货车是否形成强遮挡
4. 广告牌是否影响中景视线

### 最后测组合

1. 外星人躲在不同掩体附近时，是否仍有识别机会
2. 平民站在掩体旁时，是否不会被场景误导成目标
