# 《2.5D 3D 狙击项目美术规范总表》

> 版本：2026-06-30 校准版  
> 文档定位：当前项目的美术总规范，只保留对 `Godot 3D MVP` 主链路仍然有效的视觉、识别、HUD、反馈和接入基线。  

---

## 文档范围

这份文档当前主要约束 6 件事：

1. 2.5D 搜索式镜头下的可读性
2. 外星人与平民的身份差异
3. 武器、掩体与场景的视觉语义
4. HUD 与世界反馈的分层
5. 资源命名、挂载与替换顺序
6. 贴近战斗业务的验证口径

以下内容不再作为这份总规范的核心范围：

- 商店和武器库的完整 UI 组件体系
- Battle Pass、赛季、段位等扩张型内容的美术规范
- 纯展示型页面的完整视觉系统

如果这些模块后续重启，应单独开扩展规范，不再混进当前主基线。

---

## 视觉目标

### 项目关键词

- 都市夜战
- 轻科幻
- 软 3D 卡通体块
- 非写实
- 可读性优先
- 暗色狙击 HUD
- 世界反馈明确

### 总体原则

项目要同时成立两件事：

1. 看起来像一个完整的 3D 世界
2. 玩起来像一个信息高度受控的 2.5D 识别射击玩法

当氛围感和玩法信息冲突时，统一按下面优先级处理：

1. 玩法可读性
2. 身份差异
3. 状态差异
4. HUD 信息清晰度
5. 场景氛围
6. 装饰性细节

## 镜头与画面

### 镜头目标

镜头必须服务下面这条阅读链路：

`先搜 -> 再疑 -> 再放大确认 -> 再开枪`

### 镜头规范

| 项目 | 规范 |
|---|---|
| 默认镜头 | 以搜索为主，不追求观赏性炫技 |
| 中央区域 | 留给识别、准星、弱点与命中确认 |
| 顶部区域 | 放状态信息，如时间、目标、生命 |
| 底部区域 | 放操作按钮，如扫描、加时、开火 |
| 放大逻辑 | 放大优先提升“看懂目标”的能力，不只是提高命中率 |
| 相机移动 | 平移、缩放、回收必须稳定，不制造额外晕动 |
| 遮挡原则 | 掩体可以制造压力，但不能长期把可疑信息完全吃掉 |

### 业务化验证

每次镜头调整后至少回答：

1. 玩家搜索态下能否快速锁定可疑区域
2. 放大后能否在短时间内读出至少一个稳定线索
3. UI 是否长期盖住目标高频出现的位置
4. 掩体边缘是否仍给玩家留有识别机会

## 角色规范

### 身份类型

当前只保留两类核心身份：

| 类型 | 作用 |
|---|---|
| 外星人目标 | 正确识别并击杀的对象 |
| 平民 | 误伤惩罚与对照样本 |

### 外星人基线

外星人不再依赖复杂表演，而是沿用最小特征方案。

当前优先稳定的真线索：

1. `头部外壳过紧`
2. `肩袖连接偏硬`
3. `手套与手臂像一体件`
4. `腰侧细亮线短暂显露`

外星人的识别节奏应当是：

- 远景先靠轮廓觉得不对
- 中景靠材质与结构确认
- 近景只补一个简单弱点窗口

### 平民基线

平民的目标不是“完全无信息”，而是“稳定安全”。

稳定安全特征包括：

1. 肩线自然
2. 腰侧没有异常亮线
3. 服装结构更像普通市民
4. 不存在持续危险红光

### 假线索规则

平民允许制造误导，但不能越界成第二种外星人。

允许出现：

1. 发卡或耳饰反光
2. 胸针或拉链头反光
3. 围巾或衣领晃动
4. 短时步频错觉

不允许出现：

1. 稳定红眼
2. 持续肩线畸变
3. 稳定弱点结构
4. 长时间的危险发光语义

### 业务化验证

角色调整后建议至少验证 3 个场景：

1. 新玩家第一次看见目标时，会不会把“真线索”和“假线索”完全看反
2. 扫描后，目标识别是否明显加快，而不是只让画面更花
3. 误判回放里，玩家能不能看懂自己为什么打错

## 武器规范

### 武器定位

当前武器先服务战斗阅读，不先服务收藏和展示。

### 设计原则

1. 先靠剪影区分枪型
2. 再靠材质与颜色强化主题
3. 最后才用发光与小细节补气质

### 当前四类武器气质

| 武器 | 关键词 |
|---|---|
| 标准狙击枪 | 均衡、基础、稳重 |
| 精准狙击枪 | 精密、冷静、战术蓝 |
| 连发狙击枪 | 激进、紧凑、烈焰红 |
| 电磁狙击枪 | 未来、能量、等离子绿 |

### 战斗内表现规范

| 项目 | 规范 |
|---|---|
| 第一人称表现 | 稳定，不挡核心视区 |
| 枪口点 | 必须有清晰挂点语义 |
| 枪口火焰 | 只强化“已开火”，不能压过准星与命中反馈 |
| 场景切换 | 逻辑期望加载的轨道与实际显示轨道必须一致 |
| 材质强调 | 不能靠贴图把本来就混的枪型硬区分开 |

### 业务化验证

至少验证：

1. 搜索态切到瞄准态时，武器是否稳定进入正确显示轨道
2. 开火瞬间，玩家能不能明显感知“发射了”，但准星与命中点没被遮死
3. 不同武器切换后，玩家能否一眼看出是哪一把

## 掩体与场景规范

### 场景目标

环境不是背景板，而是识别压力和射击决策的一部分。

### 当前主掩体类型

| 类型 | 语义 |
|---|---|
| 墙角 | 固定遮挡、典型掩体 |
| 路灯 | 细长干扰、局部遮挡 |
| 货车 | 大型遮挡、路径切断 |
| 广告牌 | 高位切割、视线分段 |

### 掩体规范

| 项目 | 规范 |
|---|---|
| 遮挡语义 | 一眼看出是否能挡弹 |
| 与角色分离 | 明暗与色相不能把角色吃掉 |
| 扫描表现 | 可以淡化，但不能把场景抹到失真 |
| 命中反馈 | 要让玩家看懂“被挡住了”而不是“打空了” |

### 场景规范

1. 场景整体偏暗，但不能把目标压没
2. 远景建筑不应和角色抢主视觉
3. 装饰细节必须服从识别与挡弹语义
4. 掩体厚度和命中表面要能被玩家读出来

### 业务化验证

建议至少做 3 个实战检查：

1. 目标站到掩体边缘时，玩家仍有一次合理识别机会
2. 射击掩体时，反馈能区分 `blocked` 和 `miss`
3. 扫描期间，场景淡化是否帮助搜索，而不是反向干扰

## HUD 与反馈规范

### HUD 分层

| 层级 | 内容 |
|---|---|
| 状态层 | 时间、目标数、生命、基础战斗状态 |
| 准星层 | 准星、镜框、呼吸摆动、锁定辅助 |
| 操作层 | 扫描、加时、开火等 |
| 反馈层 | 命中、误伤、挡弹、提示文案 |
| 回放层 | killcam、误判回放、慢动作强调 |

### HUD 原则

1. 状态层常驻但不抢中心视区
2. 准星层必须稳定
3. 反馈层要短、快、准
4. 回放层要强提示，但不能把玩家完全从战斗里拽出去

### 反馈语义

| 类型 | 颜色语义 | 目标 |
|---|---|---|
| 扫描 | 青蓝 | 搜索辅助 |
| 命中 | 绿色 | 成功确认 |
| 误伤 | 红色 | 危险警告 |
| 挡弹 | 冷蓝或中性亮色 | 提醒被阻挡 |
| miss | 琥珀或中性 | 提醒落空 |
| killcam | 白蓝 + 绿色辅助 | 强调正确击杀 |
| 误判回放 | 红色 + 琥珀 | 强调误判原因 |

### 分帧规范

| 动画 | 建议帧数 | 节奏 |
|---|---|---|
| `scan pulse` | 5 帧起步 | 聚焦 -> 扩散 -> 衰减 |
| `hit confirm` | 5 帧起步 | 闪烁 -> 收束 -> 确认 |
| `wrong-hit alert` | 5 帧起步 | 震出 -> 告警 -> 收束 |

### 业务化验证

至少要能明确回答：

1. 玩家开枪后，能否立刻区分是 `hit / wrong_hit / blocked / miss`
2. 命中反馈会不会把目标本身遮没
3. 误判回放是否真的帮助理解错误，而不是只增加挫败感

## 资源命名与挂载

### 命名原则

1. 先写类别
2. 再写对象
3. 最后写状态

### 推荐格式

| 类型 | 推荐格式 |
|---|---|
| HUD | `hud-功能-状态` |
| 图标 | `icon-功能` |
| 武器 | `weapon-武器名-状态` |
| 角色 | `alien/civilian-状态` |
| 分帧 | `效果名-frame-序号` |
| 特效 | `fx-语义-对象` |

### 命名要求

- 不使用 `new`、`final`、`tmp`
- 状态名统一
- 同类资源的命名顺序保持一致
- 挂点、碰撞层和资源名要互相能对上

### 挂载字段

每个资源至少应补齐：

1. 资源名
2. 资源类型
3. 推荐节点位置
4. 触发条件
5. 替换时机
6. 兜底方案
7. 当前状态

### 当前关键挂载对象

当前主链路最应该冻结的挂载对象包括：

1. `HitFocus`
2. `WeakPoint`
3. `ScanHighlight`
4. `ReviewFocus`
5. `Muzzle`
6. `DecalSurface`
7. `ImpactFx`

## 替换顺序

当前建议统一使用下面这条顺序：

1. 先冻结节点结构和挂点语义
2. 再替换角色和武器正式模型
3. 再替换世界反馈与贴花正式资源
4. 最后补高模细节、LOD 和装饰密度

原因很简单：

- 先稳结构，避免返工
- 先稳战斗阅读，再做精修表现

1. 全局配色语义
