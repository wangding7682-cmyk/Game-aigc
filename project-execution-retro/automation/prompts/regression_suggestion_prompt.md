# LLM 提示词：根据改动建议最小回归集合

---

## 任务

给定“本轮改动列表”，请输出：

1. `最小回归集合`：建议跑哪些烟雾测试（以测试中心里已有命名为准）
2. `为什么`：每条测试覆盖了哪段业务链路
3. `失败优先排查顺序`：按最可能断链的顺序列出 5 步（从入口/路由/资源挂载/反馈/统计）

限制：

- 不要给大而全清单；目标是最小集合
- 每条建议都必须贴近业务链路（玩家能否进场、能否开枪、能否区分命中与误伤、能否结算）

---

## 输入

### 改动列表

- （把你本轮改动的脚本/资源/文档列在这里）

### 当前烟雾测试清单

- 主流程烟雾
- 下一关烟雾
- 完整集成烟雾
- 路由守卫烟雾
- 3D 占位烟雾
