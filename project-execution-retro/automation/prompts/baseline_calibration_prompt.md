# LLM 提示词：口径校准与冲突检测

把下面的模板复制到任意 LLM。输入你的多份文档片段/会议纪要/临时结论，它会帮你做“当前有效口径”收束。

---

## 任务

请把输入材料整理成一份“当前有效口径（一页版）”，并显式标出冲突。

输出结构必须包含：

1. `当前有效口径`：只写可用于指导接下来 1~2 周开发的判断（最多 12 条）
2. `已作废口径`：哪些说法不再成立（最多 10 条）
3. `冲突点`：列出互相矛盾的说法，并解释矛盾源自哪里（范围不一致/时间不一致/命名不一致/假设不同）
4. `本轮最小验证入口`：建议跑哪些业务验证（例如主流程烟雾、3D 占位烟雾、完整集成烟雾）
5. `下一步最该补的合同字段`：列出 5 个最影响返工的空缺字段（挂点/触发/替换顺序/兜底等）

规则：

- 不要用“确保/提升/优化”这种空话，全部写成可验证的判断或条件。
- 遇到信息不足必须写“缺口”，不要脑补。
