# 05 Asset Specs

## 模块目标
这个模块负责给前面的角色占位和掩体占位建立统一的：

1. 目录结构
2. 命名规则
3. 状态命名
4. 挂点命名
5. 接入字段
6. 替换顺序

它是整个 3D 美术素材工程化准备层的基础约束模块。

---

## 当前最先需要统一的内容

### 目录分层

建议统一按下面思路管理：

1. `characters`
2. `covers`
3. `feedback`
4. `decals`
5. `specs`

### 命名分层

建议统一遵循：

1. 先类别
2. 再对象
3. 最后状态

例如：

1. `alien_base_placeholder`
2. `alien_costume_placeholder`
3. `civilian_false_clue_placeholder`
4. `cover_wall_corner_placeholder`
5. `cover_street_lamp_placeholder`

---

## 当前最先需要统一的字段

每个资源后续都建议补齐：

1. `资源名`
2. `资源类别`
3. `状态`
4. `推荐节点位置`
5. `触发条件`
6. `替换时机`
7. `兜底方案`

## 当前最先需要统一的替换顺序

建议统一为：

1. 结构占位

3. 状态占位
4. 反馈占位
5. 正式资产替换

本模块后续至少要补齐：

1. `naming-rules.md`
2. `mount-fields-template.md`
3. `replacement-order.md`
4. `asset-status-template.md`

---

## 当前与主项目的关系

这个模块当前不直接改主项目资源，只负责：

- 先把规则定出来
- 先让后续所有产物有统一口径

等规则稳定后，再反向指导主项目里的真实接入。
