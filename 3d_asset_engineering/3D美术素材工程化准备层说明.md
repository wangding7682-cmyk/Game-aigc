# 3D Asset Engineering

## 目的
这个目录用于沉淀第一阶段 MVP 的 **3D 美术素材工程化准备层**。

它不是主游戏项目资源目录，也不是正式生产资产库。  
它的用途是先把现有设计素材整理成：

1. 可制作
2. 可接入
3. 可替换
4. 可测试

这样后续再逐批并入实际游戏项目时，不会把未定稿资源、命名、状态、挂点和替换顺序全部混在一起。

---

## 当前已建立的模块

### `01_character_placeholders`
角色占位工程化。

负责：

- 外星人 / 平民 MVP 占位规则
- 状态切换规则
- 弱点、假线索、扫描高亮的最小结构

### `02_cover_placeholders`
关卡与掩体占位工程化。

负责：

- 最小街区占位
- 掩体类型
- 遮挡语义
- 刷点附近的可读性

### `05_asset_specs`
资产规范与接入规则工程化。

负责：

- 目录规范
- 命名规范
- 资源状态规范
- 接入字段模板
- 替换顺序

---

### `03_feedback_rules`
战斗反馈工程化。

负责：

- `hit / wrong_hit / blocked / miss / scan / time_extend` 的结果语义
- HUD 层与世界层的反馈分工
- 反馈节奏与持续时间
- 反馈可读性检查

### `04_decal_rules`
贴花与命中痕迹工程化。

负责：

- 弹孔与命中痕迹的表面分类
- 贴花生命周期
- 贴花数量与密度限制
- 贴花可读性检查

### `06_project_mapping`
主项目映射层。

负责：

- 角色占位到主项目映射
- 掩体占位到主项目映射
- 接入字段到主项目映射

### `07_weapon_placeholders`
武器占位工程化。

负责：

- 四类武器的最小占位矩阵
- 第一人称对齐与挂点
- MVP 阶段的材质规则
- 武器可读性检查

---

## 推荐工作方式

这个目录建议按下面方式使用：

1. 先在这里定标准
2. 先在这里写模块文档
3. 先在这里确定命名、状态、挂点
4. 再逐批并入主项目

---

## 当前阶段边界

本目录只服务第一阶段 MVP，不做：

1. 正式高模生产
2. 第二阶段商业化美术扩展
3. 全量武器 / 皮肤资产生产
4. 全量正式场景模块库

---

## 当前建议推进顺序

1. 先稳定 `01_character_placeholders`
2. 再稳定 `02_cover_placeholders`
3. 同步维护 `05_asset_specs`
4. 用 `03_feedback_rules` 和 `04_decal_rules` 收口 Step 3 相关反馈规则
5. 用 `07_weapon_placeholders` 补足武器占位线
6. 最后通过 `06_project_mapping` 决定哪些产物逐批并入主项目
