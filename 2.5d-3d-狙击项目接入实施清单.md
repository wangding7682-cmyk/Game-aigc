# 《2.5D 3D 狙击项目接入实施清单》

## 目标
这份清单用于指导当前项目从“素材准备阶段”进入“正式接入前规范完善阶段”。

项目定位：

- 玩法是 `2.5D`
- 画面表现是 `3D`
- 核心体验是 `识别伪装外星人 -> 放大瞄准 -> 开火确认 -> 结算成长`

当前阶段建议：

- **先不急着全面套用素材**
- 先把“接入规范”补完整
- 再进入逐步接入

这样可以避免后面出现：

- 素材很多，但程序不知道怎么挂
- UI 好看，但 3D 场景里不可读
- 外星人 / 平民状态逻辑不统一
- 2.5D 镜头和 HUD 冲突
- 特效接进去后节奏很乱

---

## 当前素材结论
你现在已经具备了以下几类可用资产：

1. 武器卡图与武器视觉方向
2. 外星人 / 平民识别素材
3. HUD 与战斗反馈素材
4. 商店 / 武器库 UI 套件
5. 扫描 / 命中 / 误伤分帧素材
6. 环境障碍物与关卡接入示意
7. 程序可接入总板与程序挂载说明版

结论：

- **素材数量已经足够支撑接入**
- 现在最缺的不是“更多图”
- 而是“规范、挂点、顺序、测试口径”

---

## 总体实施原则

### 原则 1：先规范，后接入
先把“怎么接”定清楚，再开始大面积替换资源。

### 原则 2：先 HUD 与状态，再 3D 正式资产
因为当前玩法的核心信息首先依赖：

- HUD 可读性
- 识别状态差异
- 反馈时机

不是先依赖高模。

### 原则 3：显示资产与逻辑资产分离
每类资源都要分清：

- 视觉显示层
- 节点挂载层
- 状态驱动层
- 反馈触发层

### 原则 4：每步都要带业务测试
不做“纯看图是否好看”的测试。
每步都要回答：

- 玩家能不能看懂
- 程序能不能挂上
- 节奏有没有被破坏

---

## 实施总阶段

### 阶段 A：先补规范，不接项目逻辑
目标：把规范文档补齐，统一美术和程序口径。

### 阶段 B：做最小接入闭环
目标：只接最关键的一小段流程，先验证方向正确。

### 阶段 C：扩大接入范围
目标：把商店、武器库、战斗 HUD、角色状态接完整。

### 阶段 D：正式 3D 化与性能化
目标：从“高质量占位资产”进入“正式可交付 3D 资产”。

---

# 阶段 A：先补规范

## A1. 镜头与构图规范
这是 2.5D + 3D 项目最优先的一份规范。

### 要产出的内容

1. 默认镜头高度
2. 默认镜头倾角
3. 默认 FOV
4. 放大倍率区间
5. 目标可读距离
6. 屏幕中心安全瞄准区
7. UI 禁占区域
8. 掩体允许遮挡比例
9. 角色在前景 / 中景 / 远景的显示规则

### 为什么先做这个
因为你后面所有内容都要服从镜头：

- 外星人可不可读
- 平民会不会误导过头
- HUD 会不会挡关键目标
- 掩体有没有玩法意义

### 建议关联文件

- `狙击外星人升级版\scripts\pve\pve_battle_controller_3d.gd`
- `狙击外星人升级版\scripts\ui\ui_hud_pve.gd`
- `狙击外星人升级版\scripts\ui\ui_scope_overlay.gd`

### 业务测试

1. 放大前，玩家能否看见目标的大体位置
2. 放大后，玩家能否清楚读取“眼睛 / 肩线 / 弱点”
3. HUD 是否挡住最常见的目标站位
4. 目标在掩体边缘露出时，是否仍能识别
5. 远景下平民与外星人是否会一起糊成一团

### 达标标准

- 玩家在 3 秒内能锁定可疑目标的大致区域
- 放大后能在 1 秒内读到至少一个关键可疑特征
- UI 不遮挡主识别区域

---

## A2. 角色状态规范
这份规范决定识别玩法是否清晰。

### 需要统一的状态

#### 外星人

1. 伪装待机
2. 移动态
3. 扫描高亮态
4. 弱点开启态
5. 被命中前高危态

#### 平民

1. 普通待机
2. 普通移动
3. 假线索闪烁态
4. 误判复盘态

### 需要写清楚的内容

1. 每个状态的视觉差异点
2. 哪些差异是“真线索”
3. 哪些差异是“假线索”
4. 哪些状态由扫描触发
5. 哪些状态由行为逻辑触发
6. 哪些状态只用于教学 / 复盘

### 建议关联文件

- `狙击外星人升级版\scripts\pve\pve_target_controller_3d.gd`
- `狙击外星人升级版\scripts\pve\target_behavior_weakpoint.gd`
- `狙击外星人升级版\scripts\pve\target_behavior_moving.gd`
- `狙击外星人升级版\scripts\pve\target_behavior_factory.gd`

### 业务测试

1. 玩家第一次进入识别教学时，能否区分“真线索”和“假线索”
2. 扫描开启后，高亮是否真的提升判断速度
3. 平民假线索是否只会“短暂误导”，而不是完全误导
4. 弱点开启是否足够像“当前可击中窗口”
5. 移动态会不会让角色失去身份可读性

### 达标标准

- 真线索稳定可读
- 假线索可误导，但不会压倒真线索
- 状态切换后，角色身份仍然能被理解

---

## A3. HUD 与反馈规范
这份规范决定战斗是否“打得懂”。

### 需要统一的反馈类型

1. `scan`
2. `hit`
3. `wrong_hit`
4. `cover`
5. `miss`
6. `slowmo`
7. `killcam`
8. `misjudgment_review`

### 需要明确的内容

1. 每种反馈的颜色语义
2. 每种反馈出现的层级
3. 是否覆盖准星
4. 是否占用中央反馈区
5. 是否进入回放状态
6. 动画持续时间建议

### 建议关联文件

- `狙击外星人升级版\scripts\ui\ui_hud_pve.gd`
- `狙击外星人升级版\scripts\ui\ui_scope_overlay.gd`
- `狙击外星人升级版\scripts\pve\pve_battle_controller_3d.gd`
- `狙击外星人升级版\scripts\pve\visual_feedback.gd`

### 业务测试

1. 玩家能否区分 `hit` 和 `wrong_hit`
2. 掩体挡弹时，玩家能否理解“不是打歪，是被挡住了”
3. killcam 和 slowmo 是否打断节奏过长
4. 扫描提示是否与高亮目标同步
5. 中央反馈区是否过多堆叠信息

### 达标标准

- 0.5 秒内能读懂当前射击结果
- 中央反馈区不拥挤
- 不同结果不会误读

## A4. 商店 / 武器库组件规范
这份规范决定 UI 接入是否顺畅。

### 商店需要统一

1. tab 状态
2. 货币条状态
3. 卡片状态
4. 购买按钮状态
5. 购买成功 / 失败反馈状态

### 武器库需要统一

1. 武器列表状态
2. 皮肤列表状态
3. 详情面板结构
4. 装备按钮状态
5. 已装备 / 已解锁 / 未解锁视觉区分

### 建议关联文件

- `狙击外星人升级版\scripts\ui\ui_panel_shop.gd`
- `狙击外星人升级版\scripts\ui\ui_panel_weapon_library.gd`

### 业务测试

1. 玩家是否一眼知道当前在哪个 tab
2. 已拥有和未解锁是否容易混淆
3. 免费和可购买是否视觉差异足够
4. 装备武器和装备皮肤是否状态清楚
5. detail panel 是否足够承载武器数值信息

### 达标标准

- 状态层级清晰
- 玩家不需要读很多字就能判断当前状态
- 程序能按状态直接替换底板或按钮

## A5. 资源挂载规范
这份规范是程序接入真正的核心。

每类资产都建议统一写成这 5 项：

1. 资源名
2. 推荐节点位置
3. 触发条件
4. 替换时机
5. 兜底方案

### 示例口径

#### 战斗 HUD 资源

- `hud-scope-frame.svg`
  - 推荐节点位置：`HudLayer/root/scope_draw_layer`
  - 触发条件：`scope_visible == true`
  - 替换时机：进入放大瞄准状态
  - 兜底方案：没有正式镜框时先保留几何准星

- `hud-health-bar.svg`
  - 推荐节点位置：`HudLayer/root/top_bar`
  - 触发条件：HUD 初始化后常驻
  - 替换时机：顶部状态栏视觉升级时
  - 兜底方案：先用 Label 显示生命数值

#### 商店资源

- `shop-tab-weapon-active.svg`
  - 推荐节点位置：`ui_panel_shop/content_panel` 上层 tab 区
  - 触发条件：`current_tab == "weapon"`
  - 替换时机：完成 tab 状态资源接入后
  - 兜底方案：先用按钮文字高亮

### 业务测试

1. 程序是否能根据这份规范直接找到节点
2. 同一类资源是否都能找到统一挂载点
3. 替换时机是否和状态更新时机一致

### 达标标准

- 每个资源都有明确挂载点
- 每个资源都知道何时出现、何时替换

## 推荐实施顺序
建议严格按下面顺序推进：

1. 先定 `镜头规范`
2. 再定 `角色状态规范`
3. 再定 `HUD 与反馈规范`
4. 再定 `商店 / 武器库组件规范`
5. 再写 `资源挂载规范`
6. 然后只做一个最小战斗闭环接入
7. 验证通过后再扩到商店和武器库
8. 最后才做正式 3D 资产替换

---

## 最小必补清单
如果你想先控制投入，我建议最少先补这 6 项：

1. 2.5D 镜头规范
2. 角色状态规范
3. HUD 反馈规范
4. 商店状态规范
5. 武器库状态规范
6. 资源挂载规范

只要这 6 项补齐，你再开始正式接入，返工概率会明显降低。

---

## 当前建议结论
你现在**不建议立即全面套素材**。

更合适的路径是：

1. 先把规范补齐
2. 再做最小闭环接入
3. 通过测试后再扩大范围
4. 最后进入正式 3D 化

这样最稳，也最贴合你这个 `2.5D 玩法 + 3D 画面` 的项目。
