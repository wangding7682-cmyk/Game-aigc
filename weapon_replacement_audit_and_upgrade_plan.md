# 武器正式替换可行性审查与升级方案

## 本次结论

当前 4 把枪已经具备 **继续进入正式替换准备阶段** 的基础，但还不适合直接判定为“正式武器资产可无风险替换完成”。  
原因不是玩法链路不通，而是当前更像 **MVP 可运行枪模 + UI 方向资产 + 工程挂点已建立**，离正式量产还差一层“结构收口 + 统一挂点验收 + 第一人称遮挡验证”。

本轮同时完成了：

1. 武器正式替换可行性审查
2. 基于现有 2D 武器图反推的高完成度 3D 升级方案
3. 一套首版武器 2D 预览资产接入武器库 / 商店 / 升级页
4. 修正武器几何映射配置与电磁枪默认皮肤缺项
5. 运行 `ART_VISUAL_SMOKE` 并通过

## 一、当前 4 把枪逐项审查

审查口径基于：

1. 第一人称观察可读性
2. 缩放瞄准稳定性
3. 开火枪口点可信度
4. 材质 / 颜色是否只做辅助
5. 是否达到“可正式替换准备”最低标准

### 1. 标准狙击枪

当前状态：**通过**

- 轮廓：作为基线枪型成立，适合作为后续所有枪的挂点基准
- 缩放：当前口径最稳定，适合继续作为 `ScopeFocus` 对齐样板
- 枪口：适合作为 `Muzzle` 统一出口定义的基准枪
- 材质：中性灰可读性够用，但离正式质感还差金属层级和镜体区分

正式替换风险：

1. 风险较低
2. 主要风险不是结构，而是正式模型一旦加厚镜体或枪托，可能侵占中心视野

建议：

1. 先把它作为第一把正式替换样板枪
2. 替换时严格锁定 `Muzzle / Grip / ScopeFocus`
3. 用它反推其余 3 把枪的统一挂点口径

### 2. 精准狙击枪

当前状态：**部分通过**

- 设计方向是对的：长、更稳、更强调镜体中心
- 但本轮发现工程配置里原本 `geometry_type` 仍写成了 `rifle`
- 这会导致它在运行时不能稳定切到独立几何语义，属于正式替换前必须修掉的问题

本轮已修复：

1. `cfg_weapon_precision.tres` 的 `geometry_type` 改为 `precision`

正式替换风险：

1. 风险中等
2. 主要风险在于正式模型一旦把镜体做得更长，缩放时更容易横向偏移
3. 如果 `ScopeFocus` 不重新校准，玩家会出现“开镜后重新找中心”的问题

建议：

1. 第二把进入正式替换
2. 优先做长枪管 + 长镜体版本
3. 先测开镜中心，不要先堆材质细节

### 3. 连发狙击枪

当前状态：**部分通过**

- 方向明确，适合做更厚重的机匣和更强机械感
- 但和精准枪一样，之前工程配置里 `geometry_type` 也还是 `rifle`

本轮已修复：

1. `cfg_weapon_auto.tres` 的 `geometry_type` 改为 `auto`

正式替换风险：

1. 风险中等偏高
2. 连发枪一旦机匣做厚，很容易遮挡第一人称中心视区
3. 连续开火时如果前段体量太重，会放大“看着往前、实际像偏移”的感知问题

建议：

1. 正式模型要优先保留长枪轮廓，不要做成偏突击枪
2. 机匣可以加厚，但镜体和枪管前段必须保持清晰
3. 连续开火测试必须纳入替换前验收

### 4. 电磁狙击枪

当前状态：**部分通过**

- 方向上最有潜力做出差异化
- 但也是正式替换风险最高的一把
- 原因是它同时承担“远程精准”与“能量武器语义”两件事，最容易被发光效果带偏

本轮已修复：

1. `cfg_weapon_plasma.tres` 的 `geometry_type` 改为 `plasma`
2. 补齐了缺失的默认皮肤 `cfg_skin_plasma.tres`
3. `WeaponManager` 已纳入 `skin_plasma` 注册

正式替换风险：

1. 风险最高
2. 容易出现“结构不够，只能靠发光认枪”
3. 也容易出现能量核心抢掉瞄准中心的问题

建议：

1. 结构先行，发光后上
2. `Muzzle` 必须同时解释枪口出口和能量出口
3. 常态发光只允许在核心区小范围存在

## 三、基于现有 2D 武器图的高完成度 3D 升级方案

### 共用升级原则

1. 先保留当前第一人称挂点关系
2. 先做中模可读性，不先做高模堆细节
3. 先拉开四枪结构差异，再上材质
4. 所有正式模型必须能复用当前 `weapon_renderer_3d.gd` 挂载逻辑

### 1. 标准狙击枪升级方向

目标：

1. 成为整套枪的基线正式款
2. 强化枪托、机匣、镜体、枪管四段关系
3. 材质上做“冷金属 + 中性蓝灰辅助线”

建模重点：

1. 细化枪托过渡
2. 枪管前段保持干净
3. 镜体边框更明确，但不过度厚重

材质重点：

1. 主体为枪铁灰
2. 少量灰蓝识别线
3. 镜片只做弱反射

### 2. 精准狙击枪升级方向

目标：

1. 一眼看出“更远距、更稳定、更强调开镜”
2. 让镜体中心成为第一视觉锚点

建模重点：

1. 整体更修长
2. 枪管更细、更长
3. 镜体更长，但中心更准
4. 前段装饰要少

材质重点：

1. 冷银灰主体
2. 低饱和蓝灰辅助
3. 少量高光，不做脏污

### 3. 连发狙击枪升级方向

目标：

1. 一眼看出更偏连续压制
2. 厚重但仍保留狙击枪轮廓

建模重点：

1. 机匣加厚
2. 供弹或散热区更明显
3. 握持区更稳
4. 镜体不宜太长

材质重点：

1. 深灰主体
2. 暗红 / 暗橙灰识别区
3. 机械分层比标准枪更强

### 4. 电磁狙击枪升级方向

目标：

1. 明确不是火药枪
2. 仍然保留“远程精准武器”的稳定感

建模重点：

1. 中段加入能量核心
2. 前段加入导轨 / 线圈 / 能量出口语义
3. 镜体与能量装置分层
4. 枪口出口必须真实可信

材质重点：

1. 冷黑 / 深青主体
2. 冷青绿发光点缀
3. 发光范围只放在核心区和能量导轨关键段

## 四、推荐替换顺序

建议正式替换顺序：

1. 标准狙击枪
2. 精准狙击枪
3. 连发狙击枪
4. 电磁狙击枪

原因：

1. 标准枪最适合先固化挂点口径
2. 精准枪最能快速验证开镜中心
3. 连发枪更容易暴露遮挡问题
4. 电磁枪最复杂，应该最后替换

## 六、贴近业务的简单测试环节

建议继续按真实业务场景测，而不是只测静态界面。

### 测试 1：武器库识别测试

目标：

1. 不看名字，只看卡图，能否快速认出 4 把枪的方向差异

通过标准：

1. 至少能稳定区分标准 / 精准 / 连发 / 电磁

### 测试 2：商店购买决策测试

目标：

1. 玩家能否根据卡图 + 参数 + 稀有度快速判断“要不要买”

通过标准：

1. 武器页的预览图与参数认知一致
2. 皮肤页能明确看出它属于哪把枪

### 测试 3：升级页闭环测试

目标：

1. 玩家在结算后看到推荐升级时，能同时理解“当前用的是哪把枪”

通过标准：

1. 升级建议与当前武器卡图之间没有断层
2. 升级页不是纯数值表，而是成长决策页

### 测试 4：正式模型替换预演

目标：

1. 用一把更高完成度模型先替掉标准枪，观察视野、开镜、开火是否异常

通过标准：

1. `Muzzle` 没漂
2. `ScopeFocus` 没偏
3. 中心视野未被显著遮挡
