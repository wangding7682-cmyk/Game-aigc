这一批更偏“状态 / 套件 / 动画帧”

相较前两批，本页更强调切换关系与接入逻辑，而不是单张插画展示。你可以把这一批理解为“给程序直接接状态”和“给 UI 直接接底板”的资产集合。

扫描青蓝 命中绿色 误伤红色 移动端游戏 UI

角色状态变体

服务于目标识别、扫描高亮、弱点开启、平民假线索与移动状态，不是简单换色，而是明显状态差分。

商店 / 武器库 UI 套件

对应商店 tab、购买按钮、拥有/锁定/可购买、武器库列表卡、详情区与装备按钮，更适合直接挂到 Godot 控件上。

动画分帧素材

扫描脉冲、命中确认、误伤警告都按帧序列展示，方便直接做 HUD 动效原型或图集切片参考。

角色状态变体

这一组直接对齐目标脚本语义：外星人保留红眼脉冲、肩线异常、伪装感；平民保持冷蓝灰、自然姿态，并允许出现短暂假线索。moving 版本则通过身体重心和迈步方向强化运动感。

贴近业务的接入提示

第三批建议直接拿去做小范围原型测试，而不是只停留在静态展示：切商店 tab、切武器库装备态、触发扫描、命中目标、误伤平民，都能快速判断这批素材是否真正服务业务。

建议优先验证

  • 商店中切换 weapon / skin / item 三个 tab
  • 扫描开启后在 HUD 中叠加 alien-scan-highlight 与 scan-pulse 序列
  • 命中目标后叠加 hit-confirm 序列到成功提示或 killcam

误判相关验证

  • 对比 civilian-calm-idle 与 civilian-false-clue-glint
  • 误伤平民时播放 wrong-hit-alert 序列
  • 检查玩家是否会把假线索版本误读为真正外星人

装备页相关验证

  • 切换武器列表卡的 equipped / unlocked
  • 切换皮肤列表卡的 equipped / locked
  • 装备后检查 detail panel 与 equip button 是否形成完整操作闭环
